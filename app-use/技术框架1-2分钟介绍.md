# Reclaim Planner 技术框架 1-2 分钟介绍

Reclaim Planner 是一个本地优先的动态时间管理平台，核心不是普通日历，而是“智能调度 + 时间分析 + AI 策略建议”的一体化系统。它采用 Monorepo 架构，使用 `pnpm workspace` 和 `Turbo` 管理前端、后端、数据库、队列、集成服务和调度服务，方便后续扩展和维护。

前端位于 `apps/web`，基于 `Next.js App Router + TypeScript + Tailwind CSS + shadcn/ui + FullCalendar` 构建。用户可以在日历中查看任务、会议、习惯、专注时间，也可以手动添加计划、拖动时间块、上传时间安排文件，并通过中文界面查看时间分配分析。

后端采用 Next.js Route Handlers 作为 BFF 层，提供任务、习惯、日历同步、预约链接、AI 重排、时间安排导入、运维后台等 API。数据层使用 `Prisma + PostgreSQL`，统一抽象为 `SmartEvent`，用同一套模型表达任务、会议、习惯、缓冲时间、PTO、外部日历镜像等对象。异步任务使用 `BullMQ + Redis`，负责日历同步、Webhook、重排、统计汇总和清理任务。

调度系统分两层：第一层是本地规则引擎，处理优先级、截止时间、固定/弹性任务、锁定状态、冲突检测、缓冲时间和局部重排；第二层是独立的 `Python FastAPI + OR-Tools` 调度服务，用于更复杂的约束优化。大模型不直接替代调度器，而是负责生成“调度策略建议”和解析非结构化内容，例如课表图片、PDF、Word 或文本文件，最终仍由规则引擎和调度服务执行，保证结果可控、可解释。

系统还包含独立运维后台，用于配置 DeepSeek/OpenAI 兼容 API Key、查看调用量、健康状态和队列状态。桌面端使用 `Electron` 包装 Web 应用，启动时会尝试连接或拉起本地服务。整体优势是：结构清晰、可本地运行、可接入大模型、可解释重排、支持后续团队协作和多日历扩展。

一句话总结：Reclaim Planner 的技术路线是“Next.js 全栈应用 + Prisma 数据模型 + BullMQ 异步任务 + Python OR-Tools 调度服务 + AI 策略增强 + Electron 桌面化”，目标是把日程管理从静态记录升级为动态优化系统。
