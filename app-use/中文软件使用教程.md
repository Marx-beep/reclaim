# Reclaim 时间管理平台 中文软件使用教程

本教程用于给最终使用者直接上手软件，包含：启动、前端使用、运维后台使用、时间安排导入、常见问题。

## 1. 一键启动（Windows）

1. 双击运行：

```bat
scripts\windows\start-maintenance.cmd
```

2. 启动完成后会自动打开：
- 前端：`http://localhost:3000/`
- 运维后台：`http://localhost:3000/ops`

3. 可选命令：

```bat
scripts\windows\status-maintenance.cmd
scripts\windows\stop-maintenance.cmd
```

## 2. 前端功能说明（用户侧）

入口：`http://localhost:3000/`

### 2.1 仪表盘
- 左侧导航：仪表盘、日历、预约链接、分析、设置
- 顶部：当前日期、同步状态、同步日历
- 中央：四象限任务看板（重要且紧急 / 重要不紧急 / 紧急不重要 / 不紧急不重要）
- 右侧：智能建议、快速创建、时间安排导入

### 2.2 快速创建（已优化）
在右侧“快速创建”中：
- 添加任务（Task）
- 添加习惯（Habit）
- 生成专注时间（Focus）
- 任务和习惯都可直接自选开始/结束时间
- 任务可直接设置截止时间和优先级

### 2.3 日历里框选时间并安排任务（新）
入口：`/calendar`
- 在日历上拖拽选中时间段
- 系统自动弹出“选择任务并安排时间”
- 选择任务、填写标签（逗号分隔）后确认
- 任务会写入该时间段并自动触发排程重算

### 2.4 预约链接
入口：`/links`
- 创建预约链接
- 查看可预约时间段
- 测试预约并自动写入日历

### 2.5 分析页面
入口：`/analytics`
- 时间分配构成
- 时间趋势
- 习惯/任务投入
- 专注 vs 浅层工作
- 会议与工作-生活平衡信息

### 2.6 时间安排导入（图片/PDF/Word）
入口：仪表盘右侧“时间安排导入”

支持格式：
- 图片：`png/jpg/jpeg/webp/bmp`
- 文档：`pdf/docx/txt`

导入步骤：
1. 选择文件
2. 可选“仅预览，不写入日历”
3. 点击“识别并导入”
4. 系统显示识别条目数和新增日程数

导入接口：`/api/import/time-arrangement`

## 3. 运维后台说明（管理员侧）

入口：`http://localhost:3000/ops`

这是与前端应用分离的独立页面，用于运营监控与维护：
- 用户数、连接日历数、近期事件数
- 数据库/调度器/队列健康状态
- 手动触发同步、重排、统计、清理任务
- 查看最近重排作业与调度决策

### 3.1 Ops 登录
- 登录页：`/ops/login`
- 密码来自 `.env` 中 `OPS_ADMIN_PASSWORD`

## 4. 常见问题

### Q1：提交代码报错“请确保已在 Git 中配置 user.name 和 user.email”
执行：

```bash
git config user.name "你的GitHub用户名"
git config user.email "你的邮箱或GitHub noreply邮箱"
```

### Q2：Git 暂存区里出现大量 `.turbo` 文件
执行：

```bash
git rm -r --cached .turbo .next .runtime apps/desktop/dist
```

然后重新提交。

### Q3：页面打不开
按顺序检查：
1. `scripts\windows\status-maintenance.cmd`
2. `http://localhost:3000/` 是否返回 200
3. `http://localhost:8000/health` 是否返回正常

### Q4：队列显示 degraded
说明 Redis 版本较低，系统会自动降级，不会影响主流程（可继续使用）。

### Q5：GitHub 推送失败（网络波动或远端领先）
直接运行一键脚本（自动检测网络、自动拉取合并、自动重试推送）：

```bat
scripts\windows\git-sync.cmd
```
