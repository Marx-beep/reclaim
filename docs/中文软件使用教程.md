# Reclaim Planner 中文软件使用教程

本教程适用于把软件发给别人试用，或在另一台电脑上重新运行。

## 重要说明

`app-use` 文件夹会包含两个桌面端入口：

```text
Reclaim-Time-Manager-0.1.0-x64-setup.exe      Windows 安装包
Reclaim-Time-Manager-0.1.0-x64-portable.exe   免安装便携版
```

桌面端会打开本机的 Web 服务：

```text
http://localhost:3000
```

当前桌面端已经内置 Electron，不需要目标电脑单独安装 Electron。  
但完整动态调度能力仍依赖本地 Web/API、数据库、Redis 和 Python 调度服务。目标电脑如果没有这些服务，桌面端会尝试寻找并调用 `scripts\windows\start-maintenance.cmd`。如果要做到真正“完全免配置离线运行”，下一步需要把数据库、API 服务、Python 调度服务和运行时一起内嵌进安装包，或切换成内嵌数据库版本。

## 一、推荐使用方式

1. 把完整项目文件夹 `reclaim` 复制到目标电脑。
2. 进入项目目录。
3. 双击运行：

```bat
scripts\windows\start-maintenance.cmd
```

4. 服务启动后，打开：

```text
http://localhost:3000
```

或安装/双击：

```text
app-use\Reclaim-Time-Manager-0.1.0-x64-setup.exe
app-use\Reclaim-Time-Manager-0.1.0-x64-portable.exe
```

## 二、目标电脑首次安装要求

建议安装：

- Windows 10/11
- Git
- Node.js 18 或更高
- pnpm
- Python 3.11 或更高
- Docker Desktop

检查环境：

```bat
scripts\windows\check-env.cmd
```

如果没有 pnpm，可尝试：

```bat
corepack enable
corepack prepare pnpm@9.12.2 --activate
```

## 三、首次初始化

在项目根目录执行：

```bat
copy .env.example .env
pnpm install
docker compose -f infra\compose\docker-compose.yml up -d postgres redis
pnpm db:generate
pnpm db:migrate
pnpm db:seed
scripts\windows\start-maintenance.cmd
```

## 四、常用入口

```text
前端应用：http://localhost:3000
新版 Planner：http://localhost:3000/planner
运维后台：http://localhost:3000/ops
调度服务文档：http://localhost:8000/docs
```

## 五、主要功能

### 1. Planner 工作台

入口：

```text
http://localhost:3000/planner
```

功能包括：

- 深色左侧导航
- 周视图日历
- 任务、习惯、专注时间、会议、预约链接模块
- 右侧 AI 建议面板
- 拖动时间块
- 点击“干不下去了”触发恢复与重排
- 快速添加任务

### 2. 时间安排导入

入口：新版 Planner 左侧导航里的“日历同步”。

功能：

- 上传图片、PDF、Word、TXT、CSV 等文件
- 后端先抽取文字
- 如果配置了 DeepSeek API Key，会调用大模型解析时间安排
- 解析成功后自动填充到日历
- 如果 AI 失败，会回退本地规则解析

支持文件：

```text
图片：png、jpg、jpeg、webp、bmp
文档：pdf、doc、docx、txt、csv
```

使用步骤：

1. 打开 `http://localhost:3000/planner`
2. 点击左侧“日历同步”
3. 在“时间安排导入”卡片中选择文件
4. 选择“固定导入”或“弹性导入”
5. 可选择“仅预览，不写入日历”
6. 点击“识别并导入”
7. 系统会显示识别条目数、写入事件数和解析引擎

### 3. AI 重排

以下操作会触发重排：

- 新增任务
- 任务延迟
- 任务提前完成
- 删除任务
- 拖动时间块
- 拖动时间块到重合位置
- 点击“干不下去了”
- 手动调用重排接口

AI 不直接改数据库，而是先生成调度策略建议，再由本地规则执行。

快速添加时可以选择任务类型：

- 弹性任务：适合学习、写作、整理等可调整事项，会参与动态重排。
- 固定任务：适合课程、会议、考试、线下约定等不可挪动事项，不参与重排。

调用重排 API 时，固定任务会作为 locked/hard constraint 保留，AI 和本地规则都不能移动它。

上传时间安排文件时也可以选择固定或弹性。课程表、会议、考试建议选固定；学习计划、复习安排、待办时间块建议选弹性。

拖动任务块时允许临时重合。系统会把重合识别为冲突，然后按“固定任务优先、高优先级优先、截止时间优先、低优先级弹性任务先后移”的策略自动优化。

### 4. 时间分析

分析页可以查看：

- 专注时间
- 会议时间
- 时间分类构成
- 时间分配趋势
- 任务完成率
- 工作生活平衡

## 六、配置 DeepSeek API Key

打开运维后台：

```text
http://localhost:3000/ops
```

找到 DeepSeek/API 配置区域，填写：

```text
API Key：你的 DeepSeek Key
API URL：https://api.deepseek.com/chat/completions
Model：deepseek-chat 或 deepseek-v4-flash
```

也可以写入 `.env`：

```text
DEEPSEEK_API_KEY=你的密钥
DEEPSEEK_MODEL=deepseek-chat
DEEPSEEK_API_URL=https://api.deepseek.com/chat/completions
```

真实 Key 保存在：

```text
.runtime\llm-settings.json
```

调用用量保存在：

```text
.runtime\llm-usage.json
```

这些文件不会上传 GitHub。

## 八、常见问题

### 1. exe 打开后提示服务不可用

说明本机 `http://localhost:3000` 没启动。

先运行：

```bat
scripts\windows\start-maintenance.cmd
```

再打开 exe。

### 2. 页面还是旧样式

强制刷新：

```text
Ctrl + F5
```

或者清理缓存：

```powershell
scripts\windows\stop-maintenance.cmd
Remove-Item -Recurse -Force apps\web\.next
scripts\windows\start-maintenance.cmd
```

### 3. AI 不工作

检查：

- 是否配置 DeepSeek API Key
- API URL 是否正确
- 模型名是否正确
- 网络是否能访问 DeepSeek
- 账号是否有额度

AI 失败时，本地规则仍然可以导入和重排。

### 4. 上传文件无法识别

建议：

- 图片尽量清晰
- PDF 不要是严重扫描模糊图
- Word 中保留明确日期、星期、时间范围
- 尽量出现 `09:00-10:30 课程名` 这样的格式

### 5. 换电脑运行失败

请确认目标电脑已经复制完整项目，而不是只复制 `app-use` 文件夹。

最小可运行内容包括：

```text
apps/
packages/
services/
infra/
scripts/
package.json
pnpm-workspace.yaml
.env
```

## 九、重新生成 app-use

开发者在项目根目录运行：

```bat
scripts\windows\build-app-use.cmd
```

脚本会把最新 exe 和本教程复制到：

```text
app-use/
```
