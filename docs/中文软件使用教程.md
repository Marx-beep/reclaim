# Reclaim 时间管家（MVP）中文使用教程

本教程面向日常使用者和维护人员，包含三部分：
- 如何一键启动系统
- 如何使用前端与维护后台
- 如何打包并使用桌面 `exe`

## 1. 一键启动（Windows）

### 1.1 启动

双击或命令行执行：

```bat
scripts\windows\start-maintenance.cmd
```

脚本会自动完成：
- 读取 `.env` 配置
- 检查并启动 PostgreSQL / Redis（如果系统已安装为服务）
- 执行 Prisma 迁移（`migrate deploy`）
- 启动调度服务（FastAPI，`8000`）
- 启动 Web 服务（Next.js，`3000`）
- 自动打开：
  - 前端：`http://localhost:3000/`
  - 维护后台：`http://localhost:3000/admin`

### 1.2 查看状态

```bat
scripts\windows\status-maintenance.cmd
```

### 1.3 停止

```bat
scripts\windows\stop-maintenance.cmd
```

---

## 2. 前端功能说明（中文界面）

### 2.1 仪表盘

入口：`/`

主要区域：
- 左侧导航：`仪表盘 / 日历 / 分析 / 设置 / 维护后台`
- 顶部：当前日期、同步状态、`同步日历`按钮
- 中央：FullCalendar 时间块视图
- 右侧：`智能建议` + `快速创建`
- 抽屉：点击时间块后查看 `事件详情`

### 2.2 快速创建

在右侧 `快速创建` 面板支持：
- 添加任务
- 添加习惯
- 生成专注时间

### 2.3 分析页

入口：`/analytics`

指标包括：
- 深度工作时长
- 会议时长
- 任务完成率
- 时间利用率

### 2.4 设置页

入口：`/settings`

可配置：
- Google / Outlook 日历连接
- 时间策略（时区、工作时间、软锁/硬锁提前量）

---

## 3. 维护后台说明

入口：`/admin`

功能分区：
- 维护控制台：数据库 / 调度服务 / 队列服务状态
- 运维操作：日历同步、重排/统计/清理作业下发
- 最近重排作业
- 最近调度决策（含 reason 文本）

### 队列降级说明（非常重要）

若本机 Redis 版本低（如 `3.0.x`），页面会显示：
- `queue = degraded`
- 原因：自动禁用 BullMQ 分发

此时不会 500 崩溃，相关操作会给出可读 warning。

## 4. 打包前端桌面版 exe

### 4.1 安装依赖

在仓库根目录执行：

```bat
pnpm.cmd install
```

### 4.2 打包

```bat
pnpm.cmd --filter @reclaim/desktop dist:win
```

产物目录：

```text
apps/desktop/dist/
```

典型文件名：

```text
Reclaim-Time-Manager-0.1.0-x64.exe
```

### 4.3 使用 exe

1. 先执行 `scripts\windows\start-maintenance.cmd`
2. 再启动 `Reclaim-Time-Manager-0.1.0-x64.exe`
3. exe 将加载本地前端地址 `http://localhost:3000`

## 5. 常见问题

### Q1: 点击启动后页面打不开

排查顺序：
1. 执行 `scripts\windows\status-maintenance.cmd`
2. 检查 3000/8000 端口是否监听
3. 查看启动弹出的两个窗口（`reclaim-web` / `reclaim-scheduler`）中的报错输出

### Q2: 队列显示 degraded

这是 Redis 版本兼容保护机制，系统可继续运行。若要启用 BullMQ 队列，建议升级到 Redis 6+。

### Q3: 如何不自动打开浏览器启动？

可执行：

```bat
scripts\windows\start-maintenance.cmd -NoBrowser
```
