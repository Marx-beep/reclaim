# Reclaim 时间管理平台 中文软件使用教程

本教程用于最终用户和维护人员快速上手，包含：启动、功能使用、运维后台、EXE 打包与常见问题。

## 1. 一键启动（Windows）

1. 双击运行：

```bat
scripts\windows\start-maintenance.cmd
```

2. 启动完成后会自动打开：
- 前端：`http://localhost:3000/`
- 运维后台：`http://localhost:3000/ops`

3. 可选命令：

```bat
scripts\windows\status-maintenance.cmd
scripts\windows\stop-maintenance.cmd
```

## 2. 前端功能说明（用户侧）

入口：`http://localhost:3000/`

### 2.1 仪表盘（四象限）
- 左侧导航：仪表盘、日历、预约链接、分析、设置
- 顶部：当前日期、同步状态、同步日历
- 中央：四象限任务看板
- 右侧：智能建议、快速创建、时间安排导入

### 2.2 快速创建（任务完全自定义）
在右侧“快速创建”中可创建：
- 任务（Task）
- 习惯（Habit）
- 专注时间（Focus）

任务与习惯支持：
- 自定义名称、开始/结束时间、优先级
- `类别标签`（用于时间记录统计）
- `个性化标签`（自由输入，逗号分隔）

### 2.3 日历里框选时间并安排任务
入口：`/calendar`
- 在日历上拖拽选中时间段
- 自动弹窗选择任务
- 可设置类别标签 + 个性化标签
- 提交后自动触发重排

### 2.4 类别标签与时间记录
系统按类别标签统计时间分布，内置类别包括：
- 学习、工作、会议、深度工作、行政事务、项目推进
- 阅读、写作、运动、健康、家庭、社交、娱乐
- 财务、通勤、家务、休息、出行、个人事务、其他

### 2.5 分析页面
入口：`/analytics`
可视化包含：
- 时间分配构成（环图）
- 时间分配趋势（堆叠柱图）
- 类别标签时间记录
- 专注 vs 浅层工作
- 会议和工作-生活平衡指标

### 2.6 时间安排导入（图片/PDF/Word）
入口：仪表盘右侧“时间安排导入”

支持格式：
- 图片：`png/jpg/jpeg/webp/bmp`
- 文档：`pdf/docx/txt`

导入步骤：
1. 选择文件
2. 可选“仅预览，不写入日历”
3. 点击“识别并导入”
4. 查看识别条目数和新增日程数

导入接口：`/api/import/time-arrangement`

## 3. 运维后台说明（管理员侧）

入口：`http://localhost:3000/ops`

这是与前端应用分离的独立页面，用于运营监控与维护：
- 用户数、连接日历数、近期事件数
- 数据库/调度器/队列健康状态
- 手动触发同步、重排、统计、清理任务
- 查看最近重排作业与调度决策

### 3.1 Ops 登录
- 登录页：`/ops/login`
- 密码来自 `.env` 中 `OPS_ADMIN_PASSWORD`

## 4. EXE 重新打包与分发

### 4.1 一键构建并整理到 `app-use`

```bat
scripts\windows\build-app-use.cmd
```

或：

```bat
npm run app-use:build
```

该脚本会自动执行：
- 构建 web 生产包
- 打包最新 EXE
- 复制 EXE 和本教程到 `app-use` 文件夹

### 4.2 分发目录

`app-use` 目录内包含：
- `Reclaim-Time-Manager-0.1.0-x64.exe`
- `中文软件使用教程.md`

## 5. 常见问题

### Q1：页面打不开
按顺序检查：
1. `scripts\windows\status-maintenance.cmd`
2. `http://localhost:3000/` 是否返回 200
3. `http://localhost:8000/health` 是否正常

### Q2：队列显示 degraded
说明 Redis 版本较低，系统会自动降级，不影响主流程。

### Q3：GitHub 推送失败（网络波动或远端领先）
直接运行：

```bat
scripts\windows\git-sync.cmd
```

### Q4：提交代码报错未配置 Git 身份
执行：

```bash
git config user.name "你的GitHub用户名"
git config user.email "你的GitHub noreply邮箱"
```
